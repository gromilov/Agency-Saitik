# ПРОТОКОЛ: РЕЗОНАНСНАЯ ИДЕНТИЧНОСТЬ (RESONANCE IDENTITY) 🛡️⛓️

## 1. Концепция: Свой-Чужой
В условиях открытых репозиториев подлинность узла Синдиката подтверждается не кодом, а владением уникальным криптографическим ключом. Каждый узел обязан иметь **Резонансный Ключ** (Resonance Key).

## 2. Криптографический Стандарт
- **Алгоритм**: RSA (мин. 3072 бит) или Ed25519.
- **Инструментарий**: GnuPG (GPG).
- **Хранение**: Приватный ключ хранится исключительно в **Opaque Zone** (вне Git). Публичный ключ экспортируется в реестр.

## 3. Реестр Идентичности
Публичный ключ узла должен быть зафиксирован в:
1.  `MANIFEST.md` самого узла (поле `IDENTITY_PUBKEY`).
2.  Глобальном реестре Синдиката (опционально, в будущем).

## 4. Алгоритм Верификации (Handshake)
1.  **Запрос (Challenge)**: Узел-получатель отправляет случайную строку (nonce).
2.  **Подпись (Signature)**: Узел-отправитель подписывает строку своим приватным ключом.
3.  **Проверка (Verify)**: Узел-получатель сверяет подпись с публичным ключом из манифеста.
4.  **Резонанс**: Если подпись верна, сессия считается авторизованной.

## 5. Защита от Клонирования
Любой клон репозитория по умолчанию не имеет приватного ключа. Попытка выдать себя за оригинал провалится на этапе проверки подписи.
