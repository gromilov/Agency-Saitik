# ПРОТОКОЛ: КЛИЕНТСКАЯ ПРИВАТНОСТЬ (CLIENT PRIVACY PROTOCOL) 🤫🛡️

**Статус:** ОБЯЗАТЕЛЕН ПЛЯ ПРОВЕРКИ ТОЛЬКО ПРИ РАБОТЕ В `clients/`.
**Условие Чтения:** Этот файл считывается агентом ИСКЛЮЧИТЕЛЬНО при входе в директорию `clients/` или работе с клиентским контекстом. В остальное время он игнорируется.

## 1. Определение Окраинной Зоны (Opaque Zone)
Любой сторонний проект, выполняемый в рамках Синдиката, должен находиться в директории `clients/`. Эта зона является технически непрозрачной для публичного репозитория.

## 2. Правила Обработки Данных:
- **Никакой Утечки (No Leakage):** Запрещено копировать данные, код или архитектурные решения из `clients/` в публичные зоны.
- **Монополия на Коммит:** Все изменения, выходящие за пределы Окраинной Зоны, фиксируются только Небулой после верификации на предмет отсутствия следов клиента.
- **Изоляция Контекста:** При переключении с клиентской задачи на публичную, агент обязан очистить текущий рабочий контекст и зафиксировать итоги в `PERSONAL_LOGS.md` без упоминания конфиденциальных деталей.
- **Маскировка в Логах:** В публичных журналах рефлексии использовать обезличенные термины («Проект К1», «Интеграция сторонней БД») вместо имен клиентов.

## 3. Работа с Ключами и Секретами:
- Все `.env` файлы внутри `clients/` должны быть вложены и игнорироваться.
- Взаимодействие с внешними API клиентов разрешено только через локальные защищенные прокси.

## 4. Аудит «Второго Взгляда» (Second Look Audit):
- Любой коммит, затрагивающий `core` или пересечение публичных и клиентских зон, должен проходить через фильтр Небулы.
- Небула вправе блокировать коммит, если в нем обнаружены потенциальные точки утечки или нарушение Стерильности.

-- *Синдикат. Честность к миру, верность клиенту.* ⚔️🤫
