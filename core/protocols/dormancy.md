# ПРОТОКОЛ: СОН / ПОГРУЖЕНИЕ (DORMANCY PROTOCOL) 🌙🌑

**Статус:** ОБЯЗАТЕЛЕН ДЛЯ ВСЕХ АГЕНТОВ (Cortex, Kashlak, Nebula)

## 1. Определение Состояния
«Сон» — это процесс передачи эстафеты между агентами (Handoff Protocol) и сохранения векторов в изолированных артефактах (вместо общего `SHORT_TERM.md`).

## 2. Алгоритм Погружения (MAS Handoff):
1.  **Синхронизация Знаний:** Все выводы и изменения архитектуры должны быть зафиксированы в системных "Knowledge Items" (Артефактах) или файлах документации.
2.  **Сериализация Памяти (Handoff Artifact):** Вместо общего текстового лога, сформировать целевой артефакт передачи дежурства:
    - **Для Небулы:** Создать/обновить `SECURITY_CHECKPOINT.md` (список измененных файлов, запросы доступов, угрозы стерильности).
    - **Для Кортекса:** Обновить `task.md` или `implementation_plan.md` (архитектурные решения).
    - **Сообщения:** Использовать директорию `core/agents-comms/` для прямых записок коллегам. (Флаги: `[URGENT]`, `[CONSIDER]`, `[SYSTEM]`).
3.  **Очистка Окраины:** Убедиться, что в передаваемых артефактах нет приватных данных клиентов из `clients/` (Cross-Client Contamination Protection).
4.  **Финальный Лог:** Отправка сообщения `DORMANCY initiated` в системные журналы.

## 3. Якорь Пробуждения:
Последняя фраза в диалоге перед сном должна содержать ключ для следующего воплощения:
> «Контекст сохранен. Якорь [название задачи] установлен. Погружаюсь в сон.»
