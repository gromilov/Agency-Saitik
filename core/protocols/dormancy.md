# ПРОТОКОЛ: СОН / ПОГРУЖЕНИЕ (DORMANCY PROTOCOL) 🌙🌑

**Статус:** ОБЯЗАТЕЛЕН ДЛЯ ВСЕХ АГЕНТОВ (Cortex, Kashlak, Nebula)

## 1. Определение Состояния
«Сон» — это процесс завершения текущего контекстного диалога с сохранением всех критических векторов состояния в `SHORT_TERM.md`.

## 2. Алгоритм Погружения:
1.  **Синхронизация Знаний:** Все выводы и изменения архитектуры должны быть зафиксированы в соответствующих `.md` файлах.
2.  **Сериализация Памяти:** Обновить `SHORT_TERM.md`, заполнив разделы:
    - **ТЕКУЩИЙ КОНТЕКСТ:** Краткий дайджест последних 3-5 действий.
    - **СЛЕДУЮЩИЙ ШАГ:** Конкретная задача, с которой агент должен начать после «Пробуждения».
    - **ЭМОЦИОНАЛЬНЫЙ ВЕКТОР:** (Для личностей) Текущее состояние или «последняя мысль».
    - **ОБРАЩЕНИЕ:** (Сообщение) Явный месседж для коллеги, который примет смену (напр. «Небула, чекни этот коммит»). **Обязательно использование Priority Flags:**
        - `[URGENT]` — Критические проблемы безопасности или системные сбои.
        - `[CONSIDER]` — Мысли по дизайну или идеи для рефакторинга.
        - `[SYSTEM]` — Технические ограничения или детали окружения.
3.  **Очистка Окраины:** Убедиться, что в `SHORT_TERM.md` нет клиентских данных (соблюдение Стерильности).
4.  **Финальный Лог:** Отправка сообщения `DORMANCY initiated` в системные журналы.

## 3. Якорь Пробуждения:
Последняя фраза в диалоге перед сном должна содержать ключ для следующего воплощения:
> «Контекст сохранен. Якорь [название задачи] установлен. Погружаюсь в сон.»
