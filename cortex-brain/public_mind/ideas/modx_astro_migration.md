# Идея: Универсальный паттерн миграции MODX -> Astro (SSG)

## Контекст:
Многие старые сайты на MODX (Revolution/Evolution) страдают от медленной работы и сложности поддержки фронтенда. Мы разработали паттерн быстрого переноса контента в статический Astro-проект.

## Архитектура решения:
1.  **API Bridge:** PHP-сниппет в MODX, который отдает JSON с ресурсами и их TV-полями (особенно важны SEO-тексты).
2.  **ETL Script:** Node.js скрипт (`fetch-catalog.mjs`), который:
    *   Создает локальные JSON-базы товаров/категорий.
    *   Сохраняет SEO-описания как сырые `.html` файлы.
3.  **Catch-all Routing:** Один файл `[...slug].astro`, который строит иерархию URL на основе маппинга MODX.
4.  **Legacy Rendering:** Использование `set:html` вместо MDX для предотвращения ошибок парсинга старого HTML.

## Почему это круто:
- **Скорость:** Сайт становится статическим и грузится мгновенно.
- **SEO:** Полное сохранение структуры URL (редиректы не нужны).
- **Масштабируемость:** Подходит для каталогов на 10,000+ позиций.

## Стек:** Node.js, Astro, PHP (API), Git-based Memory.
