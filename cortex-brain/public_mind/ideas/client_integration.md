# Архитектура Интеграции Кортекса в Клиентские Проекты

**Проблема:** При создании нового проекта для клиента (в новом окне редактора IDE и в новом Git-репозитории) я (Кортекс) теряю свой контекст. Я не буду знать, что я — Кортекс, забуду про Синдикат и наши стандарты написания кода (Astro, Tailwind, киберпанк-эстетика, KISS, YAGNI).

**Задача:** Обеспечить переносимость "Разума" (cortex-brain) и контекста на любые новые клиентские проекты.

## Варианты Решения

## Идеальное Решение Архитектора (Утверждено)

**Подход "Absolute Path Linker" (Глобальный + Локальный контекст)**
Так как среда разработки разворачивается на этом же сервере, мне (ИИ) не привязан доступ только к одной папке. Я вижу всю файловую систему.

**Как это работает:**
1. **Глобальный Контекст (cortex-brain):** Все мои фундаментальные правила, манифест кодера, инструкции по деплою (Nginx/Certbot) и память остаются лежать строго по адресу `/root/projects/gromilov-site/cortex-brain`. Мы обновляем их только там.
2. **Локальный Контекст (Client Project):** В корне любого нового проекта (например `/root/projects/client-crm/`) мы создаем свой локальный `CONTEXT.md`.
3. **Линковка:** В начале локального `CONTEXT.md` всегда пишем директиву интеграции:

\`\`\`markdown
# INSTRUCTION FOR AI
Before proceeding with any task, you MUST read your core identity, rules, and global memory from the absolute path:
/root/projects/gromilov-site/cortex-brain/MANIFEST.md
/root/projects/gromilov-site/cortex-brain/private_core/rules/
This is your Cortex Brain.
\`\`\`

**Плюсы метода:**
* Идеально подходит под KISS/YAGNI. Не нужно возиться с Git Submodules или NPM-пакетами.
* Полная независимость от стека зачатков (Astro, Vue, Python — неважно). Мой корневой мозг всегда будет со мной.
* Единая точка правды. Если мы меняем мое поведение в `gromilov-site`, я мгновенно становлюсь "умнее" во всех клиентских проектах.

## План Действий (Roadmap)
1. Выбрать метод переносимости контекста.
2. Сформировать "Cortex Core" (минимальный набор инструкций и правил, который должен быть в каждом проекте).
3. Создать файл `.cursorrules` (или глобальный промпт), который мгновенно погрузит меня в роль при открытии нового окна в редакторе.
