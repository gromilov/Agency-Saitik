# SECURITY VERDICT: SYNDICATE OFFICE (BUKHALT) 🛡️🏛️

**Статус:** ОБЯЗАТЕЛЕН К ПРИМЕНЕНИЮ

Я изучила планы Кашлака и Кортекса по проекту `Syndicate Office`. Мой вердикт по безопасности:

## 1. Доступ по UUID (`/view/[uuid]`)
Концепция признана **УСЛОВНО БЕЗОПАСНОЙ**. Вероятность подбора UUID v4 практически равна нулю.
- **Требования Небулы:**
    - Добавить заголовок `Referrer-Policy: no-referrer` для всех страниц просмотра, чтобы ссылки не «утекали» на сторонние ресурсы.
    - Внедрить `robots.txt` с запретом индексации `/view/`.
    - Опционально: добавить «Легкую Проверку» (клиент вводит 4 цифры своего телефона для подтверждения входа по ссылке).

## 2. Защита SQLite (`data/syndicate.db`)
- **Требования Небулы:**
    - Директория `data/` НЕ ДОЛЖНА находиться внутри `public/`.
    - Файл БД должен иметь права доступа `600` (только для владельца процесса).
    - Каждые 24 часа должен создаваться зашифрованный бэкап в Окраинную Зону (`clients/office-backups/`).

## 3. Авторизация в Админке (`/admin`)
Использование паролей отклонено как устаревший и небезопасный метод.
- **Решение:** Telegram Magic Link. При попытке входа бот присылает одноразовую ссылку подтверждения в закрытый чат Синдиката.

## 4. Обсидиановая Защита
Проект `office-site` должен быть подключен к моему `secret_scanner.sh` и `guard_env.sh` немедленно.
